Introduction

A combobox is an input field which is a combination of a normal text input field, and a drop-down list. More technically, it uses the HTML5 tag <datalist> to suggest options for the text input.

from ipycombobox import Combobox

# An combobox without options (bascially the same as ipywidgets.Text)
Combobox()

# A combobox with a list of options
Combobox(options=[
    "Chocolate",
    "Coconut",
    "Mint",
    "Strawberry",
    "Vanilla",
])

Additionally to the behavior of the <datalist> tag, the combobox widget can optionally be set to require

# A combobox with a list of options
cbo = Combobox(
    ensure_option=True,
    options=[
        "Chocolate",
        "Coconut",
        "Mint",
        "Strawberry",
        "Vanilla",
    ]
)
cbo

When ensure_options is set to True, it will only pass back the user input value to the kernel when it is valid:

from ipywidgets import Label, jslink
label = Label()
jslink((cbo, 'value'), (label, 'value'))
label

Performance

In terms of performance, the Combobox widget is mainly limited by the browser implementation. The main performance hit is normally when changing the list of options, as this will generate a lot of DOM nodes (one <option> tag for each option). After the initial change, lookup is reasonably performant.

cbo = Combobox(ensure_option=True, options=[''.join(('Postfixed', str(i))) for i in range(100000)])
cbo
